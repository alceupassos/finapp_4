## Importação Completa de Dados F360 para Sistema BI Automatizado

A extração de todos os dados do F360 para um sistema BI automatizado requer uma **arquitetura em três camadas**: extração via API pública, armazenamento em data warehouse e visualização em ferramentas BI. Existem três estratégias principais, cada uma com vantagens específicas.

### **1. Extração via API Pública (Recomendado para Automação)**[1][2]

A API Pública da F360 oferece os principais endpoints necessários. Para configurar:

1. **Criar Chave de API** no **F360 Finanças → Webservices → Criar → API Pública da F360**
2. **Acessar Documentação**: https://documenter.getpostman.com/view/68066/Tz5m8Kcb[1]

**Endpoints Principais:**
- `GET /api/planoDeContas?cnpj=...` - Retorna todas as contas contábeis[3]
- `GET /ParcelasDeTituloPublicAPI/ListarParcelasDeTitulos` - Títulos com paginação (100 itens/página, máx. 31 dias por requisição)[2]
- `GET /api/centrosCusto?cnpj=...` - Centros de custo[3]
- `GET /api/contasBancarias?cnpj=...` - Contas bancárias[3]

A resposta inclui estrutura completa com rateios, permitindo reconstruir toda movimentação financeira.[2]

### **2. Webhooks para Captura em Tempo Real**[4][5]

Para dados em tempo real, configure um webhook que dispara automaticamente quando há novos lançamentos. O payload JSON inclui todos os campos necessários: competência, centro de custo, plano de contas e valores rateados. Use n8n ou Make para capturar esses webhooks e alimentar diretamente o data warehouse.[5][4]

### **3. Exportação em Lote via Planilhas**[6][7]

Para dados históricos completos:

1. **DRE**: F360 Finanças → DRE → Exportar (selecione "Modelos 2, 3 ou 4" para tabelas dinâmicas)[7]
2. **Contas a Pagar/Receber**: Clique em relatório → Exportar para Excel[6]
3. **Dados Detalhados**: Os relatórios exportados incluem data, empresa, meio de pagamento, competência e plano de contas[8]

### **Arquitetura Recomendada**

```
F360 API → Python Extractor → PostgreSQL Data Warehouse → Power BI/Looker
    ↑                                                      ↑
    └─────── n8n Agendado (Diário) ──────────────────────┘
```

### **Implementação Prática**

**Step 1: Script Python para Extração**

Criar um script que:
- Conecta à API F360 com chave de API
- Implementa paginação automática (máx. 31 dias por chamada)
- Transforma JSON em formato tabular
- Salva em CSV ou carrega direto no banco

**Step 2: Data Warehouse (PostgreSQL)**

Criar tabelas para:
- `f360_plano_contas` - Mapeamento de contas
- `f360_titulos` - Todas as despesas/receitas com valores
- `f360_rateios` - Detalhamento de rateios por centro de custo
- `f360_centros_custo` - Estrutura de centros

**Step 3: Automação (n8n)**

Agendar execução diária (ex: 8:00 AM):
1. Python script executa extração
2. Dados carregam no PostgreSQL
3. Tabelas de fato/dimensão sincronizam automaticamente

**Step 4: Visualização BI**

Conectar Power BI ou Looker Studio diretamente ao PostgreSQL para:
- Relatórios de DRE por período
- Análise de despesas/receitas por centro de custo
- Fluxo de caixa consolidado
- Dashboard de contas a pagar/receber por vencimento

### **Considerações Importantes**

- **Limite de 31 dias**: Cada chamada API retorna máximo 31 dias. Para histórico completo, implementar loop com incremento de datas[2]
- **Paginação**: API retorna 100 itens/página. Usar parâmetro `pagina` incrementando automaticamente[2]
- **Rate Limiting**: F360 não documenta limites, mas recomenda-se 1-2 segundos entre requisições
- **Suporte**: F360 não oferece suporte direto à API, mas documentação técnica completa está disponível[1]

Empresas e Filiais
Se o ERP Contábil que você utiliza requerer um código de empresa para os lançamentos e possuir um código de histórico padrão, siga as etapas abaixo. Caso contrário, você pode pular para a próxima seção.

Cadastro do Código da Empresa:
Acesse “Empresas e Filiais” em menu de cadastro.
Selecione a empresa e clique em “Editar”, vá para a aba “Outras Informações”.
Em dados adicionais informe o código da empresa clicando no botão "+" e escolha o layout desejado (Se não encontrar o layout desejado, entre em contato com o suporte).

Cadastro do Código Histórico Padrão:
Na mesma tela em que configurou o código, você pode cadastrar os códigos de histórico padrão do seu sistema contábil. Isso permite diferenciar as provisões, os recebimentos e pagamentos nos arquivos de Obrigações, além dos lançamentos de Transferências Bancárias.

Centro de Custo

Contas

2 - Relatórios

POST
Gerar Relatório para Contabilidade
{{URL}}/PublicRelatorioAPI/GerarRelatorio
Esse método irá gerar um relatório para a contabilidade com base nas informações do módulo F360 Finanças. Os campos obrigatórios estão em negrito:

Data: "yyyy-MM-dd"

Fim: "yyyy-MM-dd".

ModeloContabil: "provisao" ou "obrigacao". No modelo de "provisao", iremos exportar os lançamentos pela data de emissão do registro. Já no modelo de "obrigacao", iremos exportar pela data de pagamento.

ModeloRelatorio: "tradicional" ou "gerencial". O modelo "tradicional" não virá com o rateio de centros de custos, ou seja, iremos considerar apenas o CNPJ da emissão das despesas/receitas. Já no "gerencial", iremos quebrar os lançamentos pelos centros de custos gerenciais que foram informados na plataforma.

ExtensaoDeArquivo: "json" ou "csv".

CNPJEmpresas: Informar a lista de CNPJ's para a criação do relatório. Caso vazio ou nulo, o relatório irá conter as movimentações de todas as lojas.

EnviarNotificacaoPorWebbook: "true" ou "false".

URLNotificaticao: Informar a URL que deseja ser notificado após o termino da criação do relatório. Isso é importante pois o processamento do relatório é feito em background em uma fila de mensagens. Quando você informa a URL de notificação, iremos encaminhar para você um aviso assim que o relatório terminar de ser processado e estiver pronto para download.

Contas: Números externo das contas em que serão filtradas para a geração do relatório.
